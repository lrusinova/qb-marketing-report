AI Utilization in Marketing — Program Management

Luiza Rusinova  |  Marketing Program Manager, Quickbase  |  lrusinova@quickbase.com  |  June 2026

  1. Custom AI Skill (Claude.ai) — Built a custom marketing PM skill on Claude.ai encoding the full Quickbase marketing operating model: complete org structure across 3 business units and 8 teams, the What/Why/Where/How ownership framework, master brief structure with 9 sections and per-section ownership rules, Jira task conventions, and Confluence architecture. Maintained as a living knowledge base that evolves with the team. Impact: Institutional knowledge that takes a new PM 3–6 months to learn is available on demand. Every AI session starts at full context — no re-briefing.
  2. MCP Server (Model Context Protocol) — Built and configured a local MCP server connecting AI directly to Jira, Confluence, SharePoint, and Outlook via authenticated natural language. Can create and update Jira issues, run JQL queries, read SharePoint briefs, link assets to tasks, and update Confluence pages — all within a single conversation. Impact: Eliminated manual Jira navigation entirely for routine operations. What used to require opening 5 tabs and 20 minutes of data entry now takes 30 seconds of conversation.
  3. Jira Architecture Design & Build — Used AI to design and implement the full marketing Jira hierarchy from scratch: 5-level issue type structure (Strategic Project → Marketing Initiative → Epic → Task → Content Request), per-BU component taxonomy, quarter and type label system, custom field configuration per issue type, and workflow transition rules. All decisions documented and enforced via Jira permissions. Impact: A setup that would typically require an external Jira consultant and 3–4 weeks was completed in days, with cleaner architecture than most out-of-the-box implementations.
  4. H2 Program Structure — Built at Scale — Created 7 H2 Strategic Projects mapped directly from the CMO's planning slides, 20+ Marketing Initiatives across 3 BUs and 3 quarters, an epic classification framework (child vs. linked taxonomy for strategic project alignment), and 100+ epics and tasks — all with correct fields, ADF descriptions, assignees, due dates, labels, and components. Impact: ~60–80 hours of manual PM and admin work completed in a fraction of the time. A coordinator role that typically sits alongside a senior PM is no longer needed.
  5. Content Operations at Scale — Built a standardized content request template structure and applied it across 20+ Pave SEO tasks — each with SharePoint brief links, target keywords, content type, funnel stage, components, and ADF-formatted descriptions. Audited and cancelled empty placeholder issues. Created cross-BU epics and task sets (e.g. Empower 26 Content Enablement: epic + 6 tasks with owners, dates, and cross-BU classification) in minutes. Impact: Content ops work that would take a coordinator half a day is handled in a single session. Brief-to-Jira turnaround has gone from days to under an hour.
  6. Executive Reporting Dashboard — Built a live marketing status dashboard entirely with AI-generated HTML/CSS/JavaScript. Multi-tab interface covering Executive Briefing, Programs, Deliverables, and Analytics — with an H2 timeline, program status by BU, and Quickbase brand styling. Deployed to GitHub Pages as a shareable live URL. Impact: Exec-grade reporting delivered to Alys (CMO) and the ELT without a developer, BI tool, or weekly slide-building effort. Can be updated and redeployed independently in hours.
  7. Process Documentation & Ways of Working — Used AI to codify the full marketing brief system (9-section structure, ownership rules per section, brief flow from PMM signal to Jira execution), Jira task creation standards, Confluence space architecture, and communication frameworks for Slack threads and brief status tracking. All encoded in the PM skill for reuse. Impact: Processes that lived in one person's head are now documented, repeatable, and transferable — reducing single-point-of-failure risk across the marketing org.
  8. Proactive Program Auditing — AI identifies structural gaps across the Jira program (missing components, incorrect labels, orphaned epics, issues without parent links) before they surface as reporting or visibility problems. Runs as part of regular check-ins rather than as a separate audit project. Impact: Program hygiene that would otherwise require a dedicated ops review is now a built-in, continuous check — catching issues in hours rather than discovering them weeks later in a status meeting.
