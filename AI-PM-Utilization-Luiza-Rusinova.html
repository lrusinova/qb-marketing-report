AI-Powered Program Management

How Luiza Rusinova is Utilizing AI to Run Marketing Programs at Quickbase  |  June 2026

Summary: As Marketing Program Manager at Quickbase, I've built a suite of AI-powered tools and workflows that fundamentally change how marketing programs are planned, structured, and reported. What traditionally requires a team of 3–4 people (PM, Jira admin, web developer, marketing ops coordinator) I now run independently — faster, with more visibility, and with less manual overhead.

1. The AI Infrastructure I've Built

A. Marketing PM Skill (Claude.ai)

A custom AI skill encoding the full Quickbase Marketing operating model — built and maintained by me as a living knowledge base.

  • Complete org structure (3 BUs, 8 teams, 25+ named team members, reporting lines)
  • The What/Why/Where/How framework for initiative ownership across PMM, DG, and channel teams
  • 9-section master brief structure with per-section ownership rules and brief flow
  • Jira task creation rules, naming conventions, ADF format requirements
  • Confluence space structure (QBM space, page locations, hub architecture)
  • Tier definitions, brief templates, and launch process documentation

Impact: Institutional knowledge that would take a new PM 3–6 months to learn is now portable and immediately usable. Any work session starts at full context — no re-briefing required.

B. Claude Code + MCP (Model Context Protocol)

I built and connected a local MCP server that gives AI direct, authenticated access to my entire work stack via natural language:

  • Jira (Atlassian MCP): Create issues, update descriptions, set labels, components, assignees, due dates, run JQL queries, manage workflows — all via conversation
  • Microsoft 365 MCP: Search and read SharePoint files, Outlook emails, calendar — cross-platform coordination without switching tabs
  • Confluence: Create and update pages programmatically, link briefs to Jira issues

Impact: Operations that required manual Jira navigation (create epic → set fields → add ADF description → link to parent → set component → set label → tag assignee) now happen in seconds via a single conversational instruction. Bulk operations across 20+ issues that would take hours are done in minutes.

C. GitHub Pages Executive Dashboard

A live marketing reporting dashboard built entirely with AI-generated code — no developer, no BI tool, no slide deck.

  • Multi-tab interface: Executive Briefing, Programs, Deliverables, Analytics
  • H2 timeline visualization (Q2 progress + upcoming milestones)
  • Quickbase brand styling (dark QB palette)
  • Deployed to GitHub Pages — live URL, shareable with Alys and the ELT instantly

Impact: Exec-grade reporting without a developer or BI license. Can be updated and redeployed in hours when the program state changes.

2. Program Work Completed with AI

Jira Architecture — Built from Scratch

  • Issue type hierarchy: Strategic Project → Marketing Initiative → Epic → Task → Content Request — designed and implemented with correct hierarchyLevel and workflow assignments
  • Components taxonomy: Per-BU components (Pave-MKT, QB-Core-MKT, CrossBU-MKT, FF-MKT, AWE-MKT) with enforcement guidance
  • Labels taxonomy: BU labels + quarter labels (Q2-2026, Q3-2026, Q4-2026) + type labels (SEO, etc.)
  • Custom fields per issue type: Researched correct fields, multiple-assignee limitations, and field availability per type
  • Workflow configuration: Advised on and implemented "Any" incoming transitions for free status movement; identified and flagged workflow gaps
  • Jira permissions: marketing-pm group enforcing that only Luiza can create Initiatives and Epics

H2 2026 Strategic Program Structure

  • 7 Strategic Projects (MKT-12568–12574) created directly from CMO's H2 planning slides — mapped to owners, sub-goals, and cross-functional dependencies
  • Q2/Q3/Q4 Marketing Initiatives created across all 3 BUs — 20+ initiatives with correct parents, labels, and components
  • SEO initiative structure: 9 initiatives (3 BUs × 3 quarters) correctly structured and deduplicated
  • Epic classification framework: Rules for child vs. linked relationships between epics and strategic projects — enables consistent reporting

Content Operations at Scale

  • Content request templates: Standardized ADF description structure (summary format, What's needed, SEO brief, phase & ownership, footer) — applied across 20+ issues
  • Pave SEO comparison cluster: 4 tasks updated with SharePoint brief links, target keywords, content type, funnel stage, components, and labels
  • Placeholder audit: Audited and cancelled empty issues across program
  • Empower 26 Content Enablement: Epic MKT-12676 + 6 tasks created with correct assignees (Iliyan, Emma, Charlie), due dates, cross-BU classification, and ADF descriptions

Process & Documentation

  • Master brief system: 9-section structure codified with ownership per section, brief flow, and tier definitions (Tier 1 full briefs vs. Tier 2 lightweight activation notes)
  • Confluence Marketing Hub: Structure built and maintained (QBM space, Q2 folder, brief templates hub, product launches vs. internal programs taxonomy)
  • Jira + Confluence audit: Proactively identified structural misalignments (missing components/labels on Core and FF SEO initiatives)
  • Events team structuring: Epics and initiatives built for Lindsey's events team aligned to H2 structure
  • Slack thread & brief status tracking: Communication templates and status frameworks for cross-functional alignment
  • Ways of working documentation: Marketing team process docs encoding the PM framework
